Crypto-RPC Workflow

From discovery to committed results

Working with RPC endpoints follows a four-step process. Each step produces an artifact that feeds into the next. This guide shows the complete workflow from finding RPCs in the wild to committing tested results.

  1. Search

    Discovery scripts visit DEX frontends and capture network traffic. They extract RPC URLs and API keys embedded in JavaScript bundles. 

    node scripts/intercept-evm-rpcs.js base 8453

    Output written to data/evm/base-discovered.json. This file contains raw RPC candidates — URLs, authentication tokens, and source attribution.

  2. Collect

    Create input.md from the discovery results. This is a markdown table selecting which RPCs to test.

    NameURLAuthOrigin
    QuickNodehttps://…quiknode.pro/…--
    Heliushttps://rpc.helius.xyz/…-https://jup.ag

    Sources: discovery JSON, existing networks/chain/tested.md files, or manual DevTools capture. The "Origin" column is required for RPCs like Helius that validate the Referer header.

  3. Test

    The CLI runs performance and security checks against each RPC in your input file. 

    ./bin/crypto-rpc test -i input.md -o results.md
    MetricMeasuresSignificance
    RPSRequests per secondYour throughput under load
    TPSTransactions per secondChain block capacity
    Mempooltxpool_content availableCan see pending TXs
    Safe TXNo mempool accessPrivate RPC (MEV safe)
    Reading results
    "Safe TX" means the RPC does not expose pending transactions. These endpoints are safe for bundle submission without frontrunning risk.

  4. Commit

    Copy results to the permanent record and commit to version control. 

    cp results.md networks/evm/base/tested.md
git add networks/evm/base/tested.md
git commit -m "test(base): add RPCs from PancakeSwap discovery"

Common Patterns

API Key Extraction. Discovery finds keys in JS bundles. Patterns include *.quiknode.pro/*, *.alchemy.com/v2/*, rpc.ankr.com/*. These are long-lived tokens embedded in production frontend code.

Origin Requirements. Some RPCs validate the Origin header. Helius Jupiter requires Origin: https://jup.ag. Without this, requests return 403. The Origin column in your input table handles this.

Mempool Detection. The test CLI calls txpool_content. If the method exists and returns pending transactions, "Mempool" is yes. If the RPC returns "method not found" (-32601), "Mempool" is no and "Safe TX" is yes.
