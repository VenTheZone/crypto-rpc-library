Crypto-RPC Workflow

From discovery to tested results

1
SEARCH

Find RPC endpoints by scanning DEX frontends

EVM: 
node scripts/intercept-evm-rpcs.js base 8453
Solana: 
./bin/crypto-rpc discover-dex solana
Output: data/evm/base-discovered.json
Finds: RPC URLs, API keys in JS bundles
2
COLLECT

Create input.md for testing

Format:
NameURLAuthOrigin
QuickNodehttps://...--
Heliushttps://...-https://jup.ag
Source: Discovery JSON, pre-tested lists, or manual
3
TEST

Run performance & security analysis

Command: 
./bin/crypto-rpc test -i input.md -o results.md
Measures:
RPS — Requests/sec (throughput)
TPS — Transactions/sec (chain capacity)
Mempool — Pending TX visibility
Safe TX — No mempool = private
4
COMMIT

Save results to project

Copy: 
cp results.md networks/evm/base/tested.md
Commit: 
git add networks/evm/base && git commit
Results preserved for future reference

Key Concepts

Discovery Scripts

Automated scanners that visit DEX frontends and capture network traffic

input.md

Markdown table with RPCs to test — you create this from discovery results

Test Metrics

RPS = throughput, TPS = chain speed, Mempool = TX visibility

Safe TX

RPCs without mempool are safe for bundle submission (MEV protected)
